### 평가기준

    - 테스트케이스를 최대한 적게 넣어야한다.
    
    - 너무 많은 코드커버리지를 만들어서 악용하면안된다.
    
    - 조교가 숨겨놓은 최대한 많은 오류(버그)를 찾아내야한다.
    
    - assertion문장을 틀리게 넣는다던가 실수가 있으면 큰 감점.
    
    - 테스트케이스 자체에 버그가 있으면 큰 감점.(너무 많이 만들어서 assertion을 엉터리로 넣으면 안됨)
